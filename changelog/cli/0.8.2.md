## Features

- **ship `wjs` as a bin alias in @webjsdev/cli** Adds `"wjs": "bin/webjs.js"` alongside the existing `"webjs"` entry, so `npm i -g @webjsdev/cli` now puts both commands on PATH from a single install. Same behaviour from either name (both symlinks point at the same script).
  This complements the new standalone `wjs` package on npm (which also installs both symlinks). Users can pick either install path and get the canonical `webjs` and the short `wjs` together. Existing scaffold templates, `npm run dev`, AI-agent docs that reference `webjs <cmd>` continue to work unchanged.
