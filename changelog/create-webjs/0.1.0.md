## Features

- **initial release** New `npm create webjs@latest <name>` (and `npx create-webjs@latest <name>`) entry point for scaffolding a webjs app without a global `@webjsdev/cli` install. Mirrors the `create-next-app` / `create-astro` pattern, with the shorter `create-webjs` name so the `npm create` form reads as `npm create webjs@latest my-app` (matching Astro's `npm create astro@latest`).
  Supports the existing three templates (`full-stack` default, `api`, `saas`) via `--template`, and auto-installs dependencies in the new directory with the detected package manager (npm / pnpm / yarn / bun). Pass `--no-install` to opt out.
  Under the hood it dispatches to `@webjsdev/cli`'s `scaffoldApp()`, so behaviour matches `webjs create` exactly.
  Note: the earlier `create-webjs-app@0.1.0` was renamed to `create-webjs` to make the `npm create <suffix>` form read cleanly. `create-webjs-app` will be deprecated on npm with a pointer to this package.
