## Breaking

- **collapse bin map to a single `webjs` entry** 0.1.0 shipped three bins (`webjsdev`, `webjs`, `wjs`) from one package. That diluted the canonical command name and presented two unnecessary aliases as first-class. The bin map is now just `{ "webjs": "..." }`, matching `@webjsdev/cli`'s own bin map exactly.
  `webjsdev` is the unscoped npm name; `webjs` is the command. Migration note for anyone who installed `webjsdev@0.1.0` and used `wjs` or `webjsdev` as command names: replace those calls with `webjs <cmd>`. Behaviour is identical, only the command name changes.
